Welcome to Blogger
Last updated: 26/04/2025

Want to see the full blog post? https://to.linuskang.au/bloggerdocs

Blogger is a user-friendly Node.js framework designed for those who want complete control of site features and functions without compromising on simplicity. It allows you to easily set up and manage a sleek, customizable blogging site. This is the exact framework that powers this blogging site.

You can find the open-sourced code in the repository: https://github.com/linuskangsoftware/blogger

Please remember that Blogger is still in its early alpha stage! Expect a lot of things to change. A 2.0 announcement will be announced soon once I finish developing it.

How does Blogger work?

Frontend UI

The frontend provides a clean, responsive interface for users to view blog posts and navigate the site. It comes with a basic, ready-to-use layout that you can easily tweak to match your needs. Whether it's the homepage, post pages, or custom sections, the frontend is fully flexible and can be adjusted to meet your design preferences.

Backend

The backend is responsible for managing the blog's data. It handles storing, retrieving, and displaying blog posts, and it works reading .md files in the posts directory. With Blogger, you can easily create new posts, edit existing ones, and manage the structure of your site from the markdown files. The backend is built to be simple to integrate with various content management setups, giving you full control over how you structure and serve content.

That's about it. All the complex functions are handled on our side, all you need to do is follow the simple setup process and get blogging!

Certain features shown below have not yet been shipped with the production build. These extra features will be released in the next build (100-6).

Getting started with Blogger

Start by cloning the github repository:

git clone https://github.com/linuskangsoftware/blogger.git
cd blogger

Next, you'll want to install the required libaries by running the npm install command. After that, create the following directories and place them into their respective positions:

  • /posts directory inside /src
  • /blog_image_assets directory inside /src/views

Your folder structure with the added files/folders should look like this:

blogger/
└── src/
    ├── views/
    |    └── blog_image_assets/
    └── posts/

After that, head to configuration.json and edit the following settings:

port: The port that you want Blogger to run on.
host: Where you want Blogger to be accessible, leave 0.0.0.0 for whole network.
blog/blog-name: What your blog name is. This will be displayed on the blog.
blog/owner: The owners name. This will be displayed on the footer copyright notices.
blog/about-card/description: The about-me description for the home page card.

Now, you'll need a profile picture for the about-me card. Insert a png picture file inside blog_image_assets and name the picture file profile_picture.png.

Your done! Head to your terminal and start the server:

npm run dev

How to post on Blogger

Blogger works by reading .md files and displaying them statically to the client. You'll need to create markdown files in the /posts directory. One per post.

Make sure to name the .md file something easy to view, as the file name will be the path name. So 2025-04-24-hello-world.md will become yourblog.com/post/2025-04-24-hello-world

Your file should look something like this:

posts/
  └── 2025-04-24-hello-world.md

Now, you'll need content for your file. Go ahead and paste in this template below.

---
title: "My first blog post"
author: "John Doe"
date: "2025-04-26"
tags: ["New", "First post"]
badge: "John Doe"
coverImage: "/blog_image_assets/your_header_image.png"
---

Hello world!

The stuff at the top of the template is information for what your blog post is about:

  • title: The title of your blog post. This will be shown in the sidebar header of your browser and on the homepage.
  • autor: The creator of the post. This will be shown inside the post.
  • date: When the post was made. Shown both on the homepage and inside the post.
  • tags: Any relevant tags for the post. This will only be shown in the post page.
  • badge: A small badge displayed on the preview of the blog post.
  • coverImage: The header image for the blog post.

After you finish editing your blog markdown file, you'll need a header image for your blog to use the coverImage feature. Add any image to the image assets folder and change the path for coverImage to your image name.

That's it!

After configuring that, go ahead and start writing your post in markdown below where it says Hello World! and your done! You should see your post dynamically update on your browser - no need to restart the webserver.

API Reference

The following API's below are for people who want to use Blogger Server and customise their own custom frontend. These API's will be everything you'll need to achieve this.

/v1/name

Returns the blog's name

METHOD:
- GET /v1/name

HEADERS:
- Accept: application/json

RESPONSE:
{
  "name": "My blog name"
}

/v1/description

Returns the blog's name

METHOD:
- GET /v1/name

HEADERS:
- Accept: application/json

RESPONSE:
{
  "desc": "Your blog description"
}

/v1/owner

Returns the blog's owner

METHOD:
- GET /v1/owner

HEADERS:
- Accept: application/json

RESPONSE:
{
  "name": "Your blog owner name"
}

/v1/recent-posts

Returns the blog's most recent posts

METHOD:
- GET /v1/recent-posts

HEADERS:
- Accept: application/json

RESPONSE:
[
  {
    "title": "Introducing Blogger",
    "author": "Linus Kang",
    "date": "2025-04-26",
    "tags": [
      "Blog",
      "Framework",
      "Announcement",
      "Nodejs"
    ],
    "badge": "Linus Kang",
    "coverImage": "/blog_image_assets/24.04.2025-introducingblogger-header.png",
    "slug": "2025-04-24.introduction-into-blogger",
    "preview": "Blogger is a user-friendly Node.js framework..."
  }
  ...
]

/v1/all-posts

Returns all of the blog's posts

METHOD:
- GET /v1/all-posts

HEADERS:
- Accept: application/json

RESPONSE:
[
  {
    "title": "Introducing Blogger",
    "author": "Linus Kang",
    "date": "2025-04-26",
    "tags": [
      "Blog",
      "Framework",
      "Announcement",
      "Nodejs"
    ],
    "badge": "Linus Kang",
    "coverImage": "/blog_image_assets/24.04.2025-introducingblogger-header.png",
    "slug": "2025-04-24.introduction-into-blogger",
    "preview": "Blogger is a user-friendly Node.js framework..."
  }
  ...
]

/v1/post/:slug

Returns the full blog post with Markdown rendered as HTML

METHOD:
- GET /v1/post/:slug

HEADERS:
- Accept: application/json

RESPONSE:
{
  "slug": "2025-04-24.introduction-into-blogger",
  "title": "Introducing Blogger",
  "author": "Linus Kang",
  "date": "2025-04-26",
  "tags": [
    "Blog",
    "Framework",
    "Announcement",
    "Nodejs"
  ],
  "coverImage": "/blog_image_assets/your_image_header.png",
  "badge": "Linus Kang",
  "content": "<p>HTML content here...</p>",
  "rawContent": "Markdown **content** here"
}


404 RESPONSE:
{
  "error": "Post not found"
}
